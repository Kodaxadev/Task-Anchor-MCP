# Contributing to Task Anchor

Thank you for your interest in contributing to Task Anchor! As a project designed to help developers with ADHD stay focused, we value contributions that are clear, concise, and aligned with our core philosophy.

## How to Contribute

1.  **Check for Issues:** Before starting work, check the [Issue Tracker](https://github.com/Kodaxadev/Task-Anchor-MCP/issues) to see if there's an existing task or bug report.
2.  **Create a New Issue:** If you're proposing a new feature or have found a bug, please create an issue using the appropriate template.
3.  **Fork and Branch:** Fork the repository and create a branch for your work. Use a descriptive name like `feature/dopamine-layer` or `fix/drift-detection`.
4.  **Adhere to Standards:**
    *   Follow the existing code style (Python PEP 8 for the server).
    *   Ensure all new features include appropriate tests.
    *   Update documentation if necessary.
5.  **Submit a Pull Request:** When ready, submit a PR. Please ensure your PR description is clear and references any related issues.

## Design Philosophy

*   **Industrial, not infantilizing:** We build tools for professionals. Avoid "gamification" that feels like it's for children.
*   **Minimalist & Unobtrusive:** The tool should exist in the background and only intervene when necessary.
*   **Reliable:** ADHD developers rely on this for their workflow; it must be rock-solid.

Join our community discussions to share ideas or get help with your implementation.
